# Link

> **Git for understanding code.** fast, local, static code structure analysis.

`link` is a CLI tool that behaves like Git but strictly analyzes your code's structure. It parses your source code, builds an index of symbols (functions, classes, variables) and their relations (calls, imports), and provides a lightning-fast, offline, interactive graph viewer to help you understand how code connects.

Zero cloud, zero AI, zero internet required.

## Features

- **Fast & Local**: Built in Rust with Tree-sitter. Scans thousands of files in seconds. Fully offline.
- **Git-like UX**: Familiar commands (`link init`, `link list`, `link show`).
- **Visual Graphs**: Generates self-contained HTML graph visualizations that open in your default browser.
- **Cross-language**: Supports JavaScript/TypeScript, Python, Go, and Rust out of the box.

## Installation

Currently, Link is built from source. Ensure you have [Rust and Cargo](https://rustup.rs/) installed.

```bash
cargo build --release
# Move the binary to your PATH
mv target/release/link ~/.local/bin/
```

## Quick Start

1. Navigate to your project directory.
2. Initialize the index:
   ```bash
   link init
   ```
3. Show connections for a specific symbol (opens interactive graph in browser):
   ```bash
   link show my_function
   ```
4. If you modify code, update the index:
   ```bash
   link update
   ```

## Command Reference

### `link init`
Scans the current directory, indexes all supported files, and creates `.link/index.db`. Ignores `.git`, `node_modules`, `target`, `.venv`, etc., by default.

### `link show <symbol>`
Finds the target symbol and generates an interactive, directed network graph showing:
- What this symbol calls/uses (outgoing dependencies).
- What calls/uses this symbol (incoming dependencies).

Opens `.link/show.html` in your default browser.
*(Use `--json` prefix to print raw graph JSON to stdout instead).*

### `link explain <symbol>`
Text-based alternative to `link show`. Prints the incoming and outgoing connections directly to the terminal without opening a browser.

### `link list`
Lists all indexed top-level symbols (functions, classes, structs, variables) and their file locations.

### `link search <query>`
Fuzzy searches the symbol name database based on your query.

### `link update`
Fast, incremental re-indexing. Only scans files that have changed or been added/deleted since the last scan.

### `link stats`
Displays the overall health and statistics of your `.link` index database.

## Interactive Viewer & Editor Integration

The HTML graph generated by `link show` allows you to pan, zoom, and drag nodes.
**Clicking a node** will attempt to open that file at the specific line number in your code editor using a custom OS URI scheme:
`vscode://file/{absolute path}:{line}:{col}`.

*Note: Default is VS Code. Customizing the editor via `LINK_EDITOR` env var is planned for a future release.*

## Architecture & Limitations

- **Tree-sitter Parsing**: We use Tree-sitter to build a syntax tree, then run custom Scheme queries (`.scm`) to extract definitions, calls, and imports.
- **Heuristic Resolution**: Cross-file resolution is structural and name-based. It does *not* do full type inference or dynamic dispatch analysis.
  - E.g., if two files define `process()`, a call to `process()` may resolve to the one in the same file, or be marked ambiguous if no clear import/scope links them.
- **Static Only**: Link only knows what is statically determinable from syntax. Reflections, `eval`, or heavily dynamic dependency injection will not be captured.
